# Contributing to expect-us

Thanks for your interest. We keep things simple.

## Ground Rules

- Be direct. No fluff.
- Respect the project's purpose.
- No surveillance tools, backdoors, or malicious code — ever.

## How to Contribute

1. **Fork** the repository.
2. **Create a branch** — `git checkout -b feature/your-feature`
3. **Make your changes** — keep commits clean and atomic.
4. **Open a Pull Request** against `main`.

## Commit Style

Use clear, lowercase commit messages:

```
fix: correct navigation link in index.html
feat: add dark mode toggle
docs: update README members section
```

## Issues

- Use the issue templates.
- Search before opening a duplicate.
- Be specific about reproduction steps.

## Code Standards

- No minification on source files.
- HTML must pass the Proof HTML workflow.
- Keep dependencies minimal.

Questions? Open an issue.
